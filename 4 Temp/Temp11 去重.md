# 去重

在 JavaScript / TypeScript 中，**Map** 和 **Set** 都可以用来实现数组去重，但它们的适用场景和内部机制有所不同。

## 1. Set 去重

`Set` 存储唯一值，适合对**简单类型**（如字符串、数字）或**引用相同**的对象进行去重。  

- **原理**：`Set` 内部使用 `SameValueZero` 算法比较元素，对于原始类型直接比较值；对于对象，比较的是引用。
- **局限**：无法直接根据对象的某个属性（如 `date`）去重。

**示例**：对一维数组去重  

```ts
const arr = [1, 2, 2, 3];
const unique = [...new Set(arr)]; // [1,2,3]
```

## 2. Map 去重

`Map` 存储键值对，键是唯一的。适合**根据对象的某个属性去重**。  

- **原理**：将对象的指定属性作为键，整个对象作为值存入 `Map`，后面相同的键会覆盖前面的值（或根据需求保留首次出现）。
- **优势**：灵活控制去重逻辑，可保留首次或最后一次出现的对象。

**示例**：根据 `date` 属性去重（保留首次出现）  

```ts
const data = [ /* 你的数据数组 */ ];
const map = new Map();
for (const item of data) {
  if (!map.has(item.date)) {   // 保留首次出现
    map.set(item.date, item);
  }
}
const result = Array.from(map.values());
```

## 3. 主要区别

| 特性              | Set                            | Map                                        |
| ----------------- | ------------------------------ | ------------------------------------------ |
| 存储结构          | 只存值（值唯一）               | 存键值对（键唯一）                         |
| 去重依据          | 整个元素的值或引用             | 可自定义键（如对象的某个字段）             |
| 适用数据类型      | 原始类型、相同引用的对象       | 任何类型，尤其适合按属性去重的对象数组     |
| 保留首次/最后一次 | 自动去重（无法控制保留哪一个） | 可灵活控制（通过判断是否已存在或直接覆盖） |
| 性能              | 略快（结构更简单）             | 相近，多一层键的存储                       |

## 4. 何时用哪个？

- **简单数组去重**（数字、字符串） → `Set`
- **对象数组按某个字段去重** → `Map`
- **需要保留每个键对应的多个值** → `Map`（值可以是数组）

针对你之前的数据结构（对象数组按 `date` 去重），**使用 Map 是最合适的**。`Set` 无法直接实现这种去重，除非你把整个对象序列化成字符串（不可靠且低效）。
